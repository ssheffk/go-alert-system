Incident Alert System
Description: Create a system to detect and alert based on predefined metrics and logs.
Features:

Backend:
REST API for configuring alert rules.
Kafka for processing and queuing alert data.
CI/CD:
Deploy and manage services with a CI/CD pipeline.
Cloud:
Use cloud-hosted Kafka and storage.
Monitoring:
Use Prometheus to define and track alerting metrics, with Grafana for visualization. Learning Outcomes:
Setting up alert rules and notifications.
Writing robust Go services for metric processing.
Deploying and scaling on Kubernetes.
